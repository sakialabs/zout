# Docs

Documentation for Zout.

## Core Documents

### `vision.md`
**What Zout is about.**

The manifesto. The philosophy. The reason this game exists.

Read this first to understand what we're building and why.

### `tone.md`
**How Zout speaks.**

Voice principles, copy guidelines, and language rules.

Essential for anyone writing UI text, audio cues, or documentation.

### `roadmap.md`
**How we're building Zout.**

MVP definition, development phases, and what's in/out of scope.

Shows the path from concept to shippable game.

## Philosophy

All documentation follows Zout's tone:
- Short and direct
- No fluff
- Assume intelligence
- Say less

If a document doesn't add clarity, it shouldn't exist.
